# Telegram bot: контент за подписку (aiogram 3.x)

Telegram-бот по схеме **«контент за подписку»**: пользователь получает PDF‑гайд **только после подтверждения подписки** на указанный канал.

## ⚙️ Установка и запуск

### Быстрый старт (Windows)

1. Скопируйте `.env.example` → `.env` и заполните переменные
2. Запустите одним из способов:

- `run.bat` (через двойной запуск файла)
- или `run.ps1`:

```powershell
.\run.ps1
```

### Запуск из исходников (Windows / macOS / Linux)

1. Установите Python **3.11+**
2. Создайте и активируйте виртуальное окружение
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Создайте файл `.env` рядом с `main.py` и заполните по примеру `.env.example`
5. Запустите бота:

```bash
python main.py
```

### Сборка в EXE (опционально)

Если нужно собрать бота в `exe`, можно использовать PyInstaller (устанавливается отдельно).

Пример:

```bash
pip install pyinstaller
pyinstaller --onefile --name subscribe-bot main.py
```

> Примечание: при сборке убедитесь, что рядом с приложением присутствует файл `.env` и `src/guide.pdf` (или ваш реальный PDF по пути `GUIDE_PATH`).

## 📖 Документация

### Переменные окружения

Файл `.env`:

- **BOT_TOKEN**: токен Telegram-бота
- **CHANNEL_ID**: `@channel_username` или `-100...`
- **CHANNEL_LINK**: ссылка на канал, например `https://t.me/your_channel`
- **GUIDE_PATH**: путь к PDF (по умолчанию `src/guide.pdf`)
- **LOG_LEVEL**: уровень логирования (например `INFO`, `DEBUG`) — опционально

### Проверка сценариев

- **Вы подписаны** → «Проверить подписку» → бот отправляет PDF
- **Вы не подписаны** → «Проверить подписку» → бот показывает кнопку перехода в канал

> Если вы **создатель/админ** канала, то для вас подписка всегда будет считаться подтверждённой.  
> Для проверки ветки “не подписан” используйте второй аккаунт Telegram.

### Как работает проверка подписки

- Бот вызывает `get_chat_member` для `CHANNEL_ID` и пользователя.
- Подписка считается подтверждённой, если статус: `member`, `administrator` или `creator`.

### Важно для работы проверки

- Бот должен быть **администратором канала**, иначе `get_chat_member` может быть ограничен и проверка будет возвращать `False`.

### Безопасность

- **Не коммитьте** файл `.env` в репозиторий.
- Токен бота храните только в переменных окружения.
- Если токен был где-то опубликован — **перевыпустите его** в `@BotFather` командой `/revoke`.

### Частые проблемы

- **Ошибка “Файл .env найден, но он пустой (0 байт)”**: откройте `.env` и сохраните его на диск (Ctrl+S), затем запустите снова.
- **Всегда “не подписан”**: проверьте `CHANNEL_ID` и что бот добавлен **администратором** в канал.

## 🧩 Архитектура проекта

- `main.py` — точка входа, запуск polling
- `src/config.py` — загрузка `.env`, валидация, `AppConfig`
- `src/logging_setup.py` — настройка логирования
- `src/keyboards.py` — reply/inline клавиатуры
- `src/handlers/` — хендлеры команд/кнопок
- `src/utils/checks.py` — проверка подписки `is_subscribed()`
- `src/guide.pdf` — заглушка PDF (замените на ваш файл)
